模块 1：缓存
解决什么问题？
CPU直接访问主存速度过慢，开销大。所以把一些需要经常访问的数据存储在离CPU更近的缓存。
核心机制（映射、替换、更新）？
映射：把主存的数据或指令以块的形式传输到对应编号的缓存块。
就是缓存块满了，但是有新的块来映射，那么就直接用某些算法替换，比如在统计窗口里访问频率最低，LFU。以及最近使用最少的，LRU。后者用的更多。更新：当 CPU 写数据时，缓存和主存怎么保持一致，要看写策略
一致性问题怎么解决？
write back和write through，这也是更新的策略。如果是多核系统，还会涉及缓存一致性协议，但如果你现在是基础课笔记，可以先不展开。
典型实现（Intel 各级缓存）？
L1 Cache：最小、最快，通常在每个核心内部。

L2 Cache：比 L1 大，速度稍慢。

L3 Cache：更大，通常多个核心共享，速度比 L2 慢。

模块 3：超标量 + 乱序
解决什么问题？
尽管有了流水线，但是还是会有多个执行单元空闲的情况。超标量和乱序都是为了提高指令级并行（ILP），让 CPU 在一个时钟周期内尽量做更多事情。
超标量（Superscalar）：一个周期可以发射多条指令。
乱序执行（Out-of-Order）：不严格按程序顺序执行，只要操作数就绪就先执行。

发射、重命名、窗口、ROB 分别干什么？
窗口就是：
乱序处理器里暂时存放很多条待执行指令的地方。
CPU 可以从窗口里挑选“已经准备好”的指令优先执行。

ROB（Reorder Buffer，重排序缓冲区）

和流水线的关系是什么？
超标量和乱序执行不是替代流水线，而是在流水线基础上的优化策略。流水线负责把指令分阶段处理，超标量负责一个周期发射多条，乱序负责在不破坏依赖的前提下尽量让执行单元不空闲。

为什么“顺序提交”很重要？
顺序提交的重要性：

防止后面的错误结果先写入架构状态。

防止异常、分支预测错误、访存失败等情况导致程序状态混乱。

保证程序员看到的行为仍符合原始代码顺序。
