### 🧠 1. 如何定义一个AI问题
课件一开始就强调，解决任何AI问题前，必须先清晰定义。一个“良定义”的问题必须包含五个要素：

-   **初始状态**：智能体最开始的状态。
-   **可能动作**：在每个状态下，智能体可以执行的动作集合 `ACTIONS(s)`。
-   **转移模型**：描述执行某个动作后，从当前状态会转移到哪个新状态，用 `RESULT(s, a)` 函数表示。
-   **目标测试**：判断当前状态是不是我们想要达到的目标状态。
-   **路径成本函数**：给每条解决方案路径（一个动作序列）分配一个数值成本。

基于此，**解**就是从初始状态到目标状态的一个**动作序列**。其中，成本最低的那个叫**最优解**。

#### CHILD-NODE 伪代码
```
function CHILD-NODE(problem, parent, action) returns a node
    return a node with:
        STATE = problem.RESULT(parent.STATE, action)//当前在充电站 A（parent.STATE），执行 Drive(B) 动作，新状态就变成 B。RESULT 是你自己定义的转移函数。
        PARENT = parent
        ACTION = action//记录动作和父节点，这样回溯时更方便
        PATH-COST = parent.PATH-COST + problem.STEP-COST(parent.STATE, action)//新节点的路径成本 = 父节点的成本 + 这一步的成本
/*
输入：

problem：问题的定义（包含初始状态、动作集、转移模型、目标测试、成本函数）

parent：当前节点（搜索树中的一个点）

action：要执行的动作（比如“从 A 开到 B”、“充电 20%”）

输出：一个新的节点（执行动作后的状态）
*/
```

#### Tree Search and Graph Search
Tree Search: 扩展节点，将子节点加入frontier，不考虑重复状态，可能无限循环
Graph Search: 维护 explored set，避免重复扩展，只将不在 frontier 或 explored 中的节点加入

#### 性能指标中的 C* 和 ε
课件在分析 UCS 复杂度时用了两个符号：

C*：最优解的路径成本

ε：每一步的最小正成本

复杂度公式：O(b^(1 + C*/ε))

这解释了：如果每步成本差异很大，UCS 可能比 BFS 慢很多。

#### 双向搜索的“反向搜索”问题
课件提到双向搜索的一个实际难点：

“But... how do we search backwards?!”
需要能够计算状态 x 的前驱（所有能到达 x 的状态）

#### 迭代加深DFS的节点重复生成问题
课件虽然没有明说，但 IDDFS 的一个经典问题是：

同一节点在不同深度被重复生成多次

但在分支因子大的情况下，这比 BFS 的内存爆炸更可接受

这是面试中常问的“为什么用 IDDFS 而不用 BFS”的答案。

### 🛠️ 2. 搜索算法的“基础设施”与性能衡量
为了找到解，算法需要构建搜索树，课件详细介绍了支撑它的数据结构：

-   **节点结构**：每个节点需要记录四个信息：`STATE`（状态）、`PARENT`（父节点）、`ACTION`（生成它所使用的动作）、`PATH-COST`（从初始节点到该节点的路径成本）。
-   **边界**：存储待扩展节点的数据结构，通常是一个**队列**，支持`EMPTY?`、`POP`、`INSERT`三种操作。
-   **性能衡量**：评价一个搜索算法好不好，看四个指标：
    -   **完备性**：有解时，保证能找到解吗？
    -   **最优性**：找到的解是最优的吗？
    -   **时间复杂度**：要花多长时间？（通常看生成的节点数）
    -   **空间复杂度**：需要多少内存？（通常看同时存储的最大节点数）
    -   复杂度的分析常用三个参数：**b**（分支因子）、**d**（最浅目标节点的深度）、**m**（状态空间中的最大路径长度）。

### 🧭 3. 无信息搜索（盲目搜索）策略详解
这是课件后半部分的核心，也是你上传的PDF《15 Uninformed Search Strategies》的主要内容。这类策略除了问题定义本身，没有额外信息。课件详细对比了以下几种：

| 策略 | 核心思路 | 数据结构 | 完备性 | 最优性 | 时间 | 空间 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **广度优先搜索 (BFS)** | 先扩展浅层节点，一层层向外 | FIFO队列 | ✅ 是 | ✅ 是（当路径成本与深度成正比时） | O(b^d) | O(b^d) |
| **一致代价搜索 (UCS)** | 始终扩展当前路径成本最低的节点 | 优先队列 | ✅ 是（每步成本>0时） | ✅ 是 | O(b^(1+C*/e)) | O(b^(1+C*/e)) |
| **深度优先搜索 (DFS)** | 一条路走到黑，再回溯 | LIFO栈（或递归） | ❌ 否（无限状态空间） | ❌ 否 | O(b^m) | O(b*m) |
| **深度受限搜索 (DLS)** | 给DFS设置一个深度限制L | 递归 | ❌ 否（d > L时） | ❌ 否 | O(b^L) | O(b*L) |
| **迭代加深DFS (IDDFS)** | 以递增的深度限制反复运行DLS | 调用DLS | ✅ 是 | ✅ 是（与BFS条件相同） | O(b^d) | O(b*d) |
| **双向搜索** | 从初始状态和目标状态同时进行BFS | 两个FIFO队列 | ✅ 是 | ✅ 是（当BFS各自最优时） | O(b^(d/2)) | O(b^(d/2)) |

课件用一个具体例子震撼地展示了BFS的空间爆炸问题：假设分支因子 b=10，当目标深度 d=12 时，BFS需要13天时间和1PB内存。这直接说明了为什么空间效率高的DFS、IDDFS在现实中很重要。
