# 笔记
## static
+ 静态的对象一旦创建生命贯穿整个文件
+ 类内的静态成员变量只会初始化一次，也就是在对象创建的那个瞬间。
+ 在C++里，类的静态变量不能直接在类里面赋值，你必须在类的外面，专门写一行代码来给它设定初始值。而且，在写这行代码的时候，你要明确告诉电脑这是哪个类的变量。
## 虚表
+ Base *pt = new Derived(); 
    pt->fun1(); // 这行代码的底层发生了什么？
    第一步：识别“宝藏入口”（虚函数）
    程序发现 fun1 是一个虚函数。这意味着它不能简单粗暴地调用 Base::fun1()，而是要启动一套“动态查找”机制。

    第二步：找到对象的“藏宝图”（vptr 和 虚函数表）
    C++编译器会给每个包含虚函数的类（如 Base 和 Derived）自动生成一张虚函数表（Virtual Table，简称vtable）。这张表就是一个数组，里面存放了该类所有虚函数的真实地址。

    同时，每个由这类创建出来的对象（比如 pt 指向的那个 Derived 对象）都会被编译器安插一个隐藏的指针，叫虚指针（vptr）。这个 vptr 就指向它所属类的虚函数表。

    所以，pt->vptr 的意思就是：顺着 pt 这个指针，找到它指向的那个真实对象（Derived对象），然后找到这个对象里的隐藏指针 vptr。由于这个对象是 Derived 类型的，它的 vptr 自然指向 Derived 类的虚函数表。

    第三步：查阅“藏宝图”（查找函数地址）
    现在程序拿到了 Derived 的虚函数表，它知道 fun1 在表中的第几个位置，于是直接去那个位置查。

    它查到的地址是什么？是 Derived::fun1() 的地址！（因为 Derived 重写了 fun1，所以它的虚函数表里 fun1 的位置放的是它自己的函数地址。如果 Derived 没重写，那这个位置放的会是 Base::fun1() 的地址。）
## volatile
+ 是什么： 一个关键字，用来禁止编译器优化。

+ 怎么用： 告诉CPU，每次都用变量都必须从内存原始地址读取，不要用缓存里的副本。

+ 运用场景：
    操作硬件寄存器（如传感器数据）。
    在中断服务程序里修改的全局变量。
    多线程中（不推荐，仅作为了解，建议用atomic）。
