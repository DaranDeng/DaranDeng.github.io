## 问题
+ 没有区分大小写 & 输入非法字符，缺失检测输入的关键环节，直接 scanf("%c",&ch); 拿到回车、数字、符号都算一次猜测。
  怎么验证输入的只是字母呢，两种方法：
  + `if((ch >= 'A' && ch <= 'Z') || (ch >= 'a' && ch <= 'z'))`
  + `#include<ctype.h> isalpha()`
+ 没有初始化显示区
第一次进入循环前，玩家完全不知道单词有多长。
→ 信息不完整。
+ 你用 i 当“当前要猜第几个字母”，玩家必须按顺序一个一个猜对才算数。
标准 Hangman 是“任意顺序猜字母，猜中就显示所有该字母的位置”。
+ 怎么将数组全部初始化为0，两种方法：
  +   `memset(arr, 0, sizeof(arr));`
  +   `int arr[5] = {0};`话说我为什么这样搞失败了？查了一下，发现`int len = 100;
int isGuessed[len] = {0};  // 错误！len 不是 const`
这在 C99 之前是非法的，即使 C99 支持变长数组（VLA，VLA意味着数组大小是变量），也不能用 VLA 做静态初始化！
变长数组（VLA）不允许用 {0} 整体初始化，只能一个个赋值。
+ 数组作为形参的时候，如果要修改就用指针。否则没有必要。字符串字面量是只读的内存。但字符数组可改。通过字符串字面量的指针来改变字面量是不允许的。比如这个`char *str1 = "geeks"`是改不了的哪怕传递str1。
+ 记得手动添加'\0'，没有哪个“内嵌函数”会自动停止而不加 '\0'
你记住的“自动停止”其实是：
字符串字面量 "abc" 编译器自动加 '\0'。`char str[] = "hello";  // 编译器自动在末尾加 '\0'`
大多数字符串函数（如 strcpy, scanf, fgets）会自动加上 '\0'，printf不会
+ 在函数的函数头写的形参 isGuessed[]与*isGuessed,char lines[2][len+1]与char **lines有什么不同吗？
  前者是等价的。后者的问题关键在于char lines[2][len+1]的数组名lines是不是指针的指针？不是，是指向数组的指针，不要混淆数组与数组名，lines+1移动len+1个char，而char **line，line+1移动一个指针。
